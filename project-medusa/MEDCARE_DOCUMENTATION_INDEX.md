## üéØ Key Concepts Explained

### MEDUSA vs MedCare
| Aspect | MEDUSA | MedCare EHR |
|--------|--------|------------|
| **Purpose** | Analyzer/Attacker | Target/Lab |
| **Role** | Pentesting platform | Vulnerable infrastructure |
| **Services** | 5 (CLI, Frontend, Backend, DB, Cache, Neo4j) | 7 (API, Database, SSH, FTP, LDAP, Log Viewer, Workstation) |
| **Networks** | medusa-dmz | healthcare-dmz + healthcare-internal |
| **Note** | PHP ehr-webapp is disabled, replaced by Next.js medusa-frontend |

### Service Naming
- **MEDUSA services:** `medusa-*` (e.g., medusa-frontend, medusa-backend)
- **MedCare services:** `ehr-*` (e.g., ehr-api, ehr-database) or descriptive names (ssh-server, ftp-server)
- **Container names:** Always `medusa_*` (e.g., medusa_ehr_api, medusa_frontend)
- **Note:** PHP ehr-webapp service is disabled; medusa-frontend serves as the primary EHR frontend

### Networks
- **medusa-dmz:** 172.22.0.0/24 - MEDUSA platform
- **healthcare-dmz:** 172.20.0.0/24 - Public-facing MedCare services
- **healthcare-internal:** 172.21.0.0/24 - Backend MedCare services
